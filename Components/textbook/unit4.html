Unit 4: Linked Lists

1. Traversing and Searching

Linked lists are dynamic structures where elements (nodes) are connected via pointers. Traversal means visiting each node sequentially, while searching involves comparing node data until the target is found.
Complexity: O(n).

2. Insertion

  • Beginning: Insert a new node at the head.
  • Given node: Insert after a specific node.
  • Sorted list: Insert while maintaining order.

3. Deletion

Deletion requires adjusting pointers to bypass the node being removed. Cases include deleting the head, a middle node, or the last node.
Complexity: O(n) for search + O(1) for pointer adjustment.

4. Header Linked Lists

  • Grounded: Ends with a null pointer.
  • Circular: Last node points back to the first, enabling continuous traversal.

5. Applications

Linked lists are used in dynamic memory allocation, implementing stacks and queues, polynomial representation, and adjacency lists in graphs.
