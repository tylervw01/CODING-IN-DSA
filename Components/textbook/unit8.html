Unit 8: Sorting and Searching

1. Sorting Algorithms

  • Bubble Sort: Repeatedly swap adjacent elements. O(n²).
  • Selection Sort: Select minimum element and place it in order. O(n²).
  • Insertion Sort: Insert elements into their correct position. O(n²), but O(n) best case.
  • Merge Sort: Divide and conquer, merge sorted halves. O(n log n).
  • Quick Sort: Partition around a pivot, recursively sort. Average O(n log n), worst O(n²).

2. Searching Algorithms

  • Linear Search: Scan each element sequentially. O(n).
  • Binary Search: Requires sorted array. Divide and conquer. O(log n).
  • Hashing: Use hash functions for constant-time search. Average O(1).

3. Complexity Comparison

Efficient algorithms (Merge Sort, Quick Sort, Binary Search, Hashing) outperform naive ones (Bubble, Linear). Choosing the right algorithm depends on data size, order, and constraints.

4. Applications

  • Database indexing and query optimization.
  • Search engines and information retrieval.
  • Sorting records in management systems.
  • Hash tables in compilers and operating systems.
