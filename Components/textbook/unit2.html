Unit 2: Data Structure for String and Pattern Matching Algorithm

1. String Storage Methods

  • Fixed-length: Strings stored with a predefined size.
  • Variable-length: Strings stored with dynamic size allocation.
  • Linked storage: Strings stored as linked nodes for flexibility.

2. String Operations

  • Substring: Extracting part of a string.
  • Indexing: Accessing characters by position.
  • Concatenation: Joining two or more strings together.

3. Pattern Matching Algorithms

  • Naive approach: Check each position in the text for a match.
  • Table-driven (e.g., KMP): Preprocess the pattern to skip unnecessary comparisons.

4. Applications

String and pattern matching algorithms are widely used in text editors, search engines, DNA sequence analysis, and data validation systems.
