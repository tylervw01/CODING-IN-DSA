Unit 6: Trees

1. Basic Concepts

A tree is a non-linear hierarchical data structure consisting of nodes. The top node is the root, and each node may have child nodes. Trees are widely used for representing hierarchical relationships.

2. Binary Trees

A binary tree is a tree where each node has at most two children: left and right. Traversals include:

  • Preorder: Root → Left → Right
  • Inorder: Left → Root → Right
  • Postorder: Left → Right → Root

3. Binary Search Trees (BST)

A BST is a binary tree where the left child contains values less than the root, and the right child contains values greater than the root.
Applications: Searching, sorting, dynamic sets.
Complexity: O(log n) average, O(n) worst case (unbalanced).

4. Balanced Trees

Balanced trees (e.g., AVL, Red-Black Trees) maintain height balance to ensure efficient operations. They guarantee O(log n) for insertion, deletion, and search.

5. Tree Applications

  • Expression trees in compilers.
  • File system hierarchy.
  • Decision trees in AI.
  • Routing tables in networks.
